## Introduction

Year 2 begins where Year 1 finished, but moves faster. By now you have met sequences, inputs, outputs, loops, conditionals, and debugging. This first lesson is a chance to revisit the three most fundamental ideas and make sure they are solid before you build on them. Understanding inputs, sequences, and outputs as a connected system is the foundation for everything in this year.

## What We Will Learn

- What sequences, inputs, and outputs are and how to explain each one clearly.
- How these three ideas connect to form one complete system.
- How to identify all three in a new context.
- Where the gaps are in your understanding so you know what to focus on.

## Core Explanation

**Sequence**

A sequence is a list of steps in a specific order. Each step runs once, in turn, from top to bottom. Order matters because many steps depend on previous ones.

**Input**

An input is information given to a system from outside. For a computer, inputs come from keyboards, mice, touchscreens, microphones, and sensors. The input triggers something to happen, or provides data the program needs to run.

**Output**

An output is the result that a system sends back to the world. For a computer, outputs appear on screens, come from speakers, print on paper, or cause a physical device to move or light up.

**How they connect**

These three things work together. A user provides input. The program runs a sequence using that input. The sequence produces an output. Then the user sees the output and may provide new input, starting the cycle again.

Think about using a calculator. You press keys (input). The calculator runs a sequence of operations (sequence). The answer appears on the screen (output). You see the answer and press more keys (new input). The cycle continues.

This input-sequence-output structure applies to almost every computing system that exists, from a light switch to a space probe.

## Activity

**System Analysis** (28 minutes)

Part 1: Warm-up recall (8 minutes)

Step 1 (4 minutes): Rapid-fire class recall. Teacher points to a student and names a device. The student must say one input and one output for that device. Devices: phone, microwave, ATM, automatic door, video game, traffic light.

Step 2 (4 minutes): Teacher draws a simple blank diagram on the board: a box labelled INPUT, an arrow to a box labelled SEQUENCE, an arrow to a box labelled OUTPUT. Fill it in together for one example: a search engine. Input: the words you type. Sequence: search through millions of pages and rank results. Output: a page of matching links.

Part 2: Diagram analysis (15 minutes)

Step 3 (5 minutes): Give each pair a diagram or description of an unfamiliar system. Options: a vending machine, an automatic hand dryer, a ticket barrier at a station, a smoke alarm, a car's reversing sensor. Pairs must label the input, the sequence (even if simplified), and the output.

Step 4 (5 minutes): Each pair presents their analysis in two sentences. Class checks: are the three parts distinct? Is anything labelled in the wrong category?

Step 5 (5 minutes): Address two common confusions. First: is a button an input or an output? (Input. It gives information to the system.) Second: can one device have multiple inputs and multiple outputs? (Yes. A phone has dozens of both.) Show one example.

Part 3: Independent description (5 minutes)

Step 6 (5 minutes): Each student chooses any system from their daily life and writes a three-part description: "The input is... The sequence does... The output is..."

## Real-World Connection

Every piece of software ever written takes some form of input, runs a sequence of instructions, and produces some form of output. A website, a robot arm in a factory, a medical scanner, a music streaming app: all follow this same structure. Understanding input-sequence-output is not just a school exercise. It is the mental model that engineers use when they design new systems.

## Check for Understanding

1. A fire alarm detects smoke and sounds a siren. Identify the input, the sequence (briefly), and the output.
2. A student says "the screen is the input because I look at it." What is wrong with this, and how would you correct it?
3. Name one system where the output of one cycle becomes the input of the next.

## Extension and Differentiation

**For students who need more support:** Use only familiar devices: a light switch (input: flip switch, sequence: electricity flows, output: light turns on) and a television remote. Draw the diagram together for each one.

**For students who want a challenge:** Find a system where the sequence includes a conditional and a loop. Describe all five parts: input, condition, action-if-true, loop, output. A washing machine is a good example.

## Summary

- A sequence is a set of ordered steps. An input gives the system information. An output is the result the system produces.
- These three parts work together in a cycle: input triggers the sequence, the sequence produces the output.
- Almost every computing system uses this structure, from a simple calculator to a complex robot.
- Identifying these three parts in new systems is a transferable skill that applies to all of computing.
